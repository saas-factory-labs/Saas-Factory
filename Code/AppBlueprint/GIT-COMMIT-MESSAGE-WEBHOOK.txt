## Features Added
- **Stripe Webhook Service** with signature verification using official Stripe.net library
- **Idempotency checking** to prevent duplicate event processing via unique (EventId + Source) constraint
- **Event storage** for auditing, replay, and compliance with WebhookEventEntity aggregate
- **Automatic retry logic** for failed events with configurable max retry attempts
- **Tenant isolation** via automatic metadata extraction from Stripe objects
- **Production-ready API endpoint** at `/api/v1/webhooks/stripe` with proper error handling
- **Comprehensive README** with setup guide, testing instructions, and troubleshooting

## Architecture (Clean Architecture)
- **Domain Layer** (`AppBlueprint.Domain`)
  - `WebhookEventEntity` - Event aggregate with status tracking (Pending, Processed, Failed, PermanentlyFailed)
  - `IWebhookEventRepository` - Repository interface for event persistence
  
- **Application Layer** (`AppBlueprint.Application`)
  - `IStripeWebhookService` - Service interface for webhook processing
  - `WebhookProcessingResult` - Result type for webhook operations
  - `WebhookEventDetails` - DTO for querying webhook events
  
- **Infrastructure Layer** (`AppBlueprint.Infrastructure`)
  - `StripeWebhookService` - Service implementation with signature verification
  - `WebhookEventRepository` - EF Core repository implementation
  - `WebhookEventEntityConfiguration` - Database configuration with indexes
  - `StripeWebhookServiceExtensions` - DI registration helper
  
- **Presentation Layer** (`AppBlueprint.Presentation.ApiModule`)
  - `StripeWebhookController` - Public POST endpoint for Stripe webhooks

## Supported Stripe Events (Extensible)
- **Payment:** `payment_intent.succeeded`, `payment_intent.payment_failed`
- **Customer:** `customer.created`, `customer.updated`, `customer.deleted`
- **Subscription:** `customer.subscription.created/updated/deleted`
- **Invoice:** `invoice.paid`, `invoice.payment_failed`
- **Checkout:** `checkout.session.completed`

## Security Features
- ✅ Signature verification using Stripe webhook secrets (whsec_...)
- ✅ Idempotency via database unique constraint on (EventId, Source)
- ✅ Event audit trail for compliance and debugging
- ✅ Automatic retry with exponential backoff (max 3 attempts)
- ✅ Tenant isolation via metadata extraction (tenant_id in Stripe object metadata)

## Configuration
Environment variables (multiple naming conventions supported):
```bash
# Option 1: Recommended
STRIPE_WEBHOOK_SECRET=whsec_YourSecretHere

# Option 2: Legacy dotnet format
Stripe:WebhookSecret=whsec_YourSecretHere
```

## Usage Example
```csharp
// Program.cs
builder.Services.AddStripeWebhookService(builder.Configuration);

// Stripe automatically POSTs to: POST /api/v1/webhooks/stripe
// Response:
// {
//   "message": "Webhook processed successfully",
//   "eventId": "evt_1234567890",
//   "eventType": "payment_intent.succeeded",
//   "wasDuplicate": false
// }
```

## Testing with Stripe CLI
```bash
# Forward webhooks to local endpoint
stripe listen --forward-to http://localhost:5000/api/v1/webhooks/stripe

# Trigger test events
stripe trigger payment_intent.succeeded
stripe trigger customer.subscription.created
```

## Database Schema
```sql
CREATE TABLE "WebhookEvents" (
    "Id" VARCHAR(40) PRIMARY KEY,          -- whevt_01ABCD...
    "EventId" VARCHAR(255) NOT NULL,       -- evt_1234567890
    "EventType" VARCHAR(255) NOT NULL,     -- payment_intent.succeeded
    "Source" VARCHAR(50) NOT NULL,         -- stripe
    "Payload" TEXT NOT NULL,               -- Raw JSON
    "TenantId" VARCHAR(40) NOT NULL,       -- tnt_01ABCD... (or empty)
    "Status" VARCHAR(50) NOT NULL,         -- Pending/Processed/Failed
    "RetryCount" INT NOT NULL DEFAULT 0,
    "ErrorMessage" VARCHAR(2000),
    "ReceivedAt" TIMESTAMP NOT NULL,
    "ProcessedAt" TIMESTAMP,
    CONSTRAINT "UX_WebhookEvents_EventId_Source" UNIQUE ("EventId", "Source")
);
```
