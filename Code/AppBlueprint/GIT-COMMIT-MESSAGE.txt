## Features Added
- **Email Template Service** with RazorLight engine for compile-time safe templates
- **Strongly-typed email models** for 7 common scenarios (Welcome, Password Reset, Order Confirmation, Invoice, Booking, Weekly Digest, Admin Notifications)
- **Framework-provided generic templates** with app-level override capability
- **Interactive demo portal** at `/email-templates` with live preview and test email sending
- **Resend email service integration** with proper API configuration following official docs
- **Multiple environment variable naming conventions** (APPBLUEPRINT_RESEND_*, RESEND_*, APPBLUEPRINT_Resend__*)

## Architecture
- **IEmailTemplateService** interface in Application layer
- **RazorEmailTemplateService** implementation using RazorLight + Resend
- **Template override pattern**: App `/Templates/` folder â†’ Framework embedded resources
- **Email-safe HTML templates** with table layouts and inline styles
- **Optional Resend dependency** using factory pattern for graceful degradation

## Email Models
1. WelcomeEmailModel - User onboarding with optional activation link
2. PasswordResetEmailModel - Password reset with expiring token link
3. OrderConfirmationEmailModel - E-commerce order confirmation
4. InvoiceEmailModel - Invoice notifications with payment link
5. BookingConfirmationEmailModel - Reservation/booking confirmations
6. WeeklyDigestEmailModel - Periodic summary emails with activity list
7. AdminNotificationEmailModel - System alerts for administrators

## Demo Portal Features
- Template selection dropdown (Welcome, Password Reset, Order Confirmation)
- Dynamic forms for template-specific data
- Live HTML preview with iframe rendering
- HTML source view toggle
- Test email sending to any address
- Responsive flexbox layout
- Uses configured Resend sender domain from environment variables

## Configuration
Supports 3 naming conventions (priority order):
1. `APPBLUEPRINT_RESEND_APIKEY` / `APPBLUEPRINT_RESEND_FROMEMAIL` / `APPBLUEPRINT_RESEND_FROMNAME` (new standard)
2. `RESEND_API_KEY` / `RESEND_FROM_EMAIL` / `RESEND_FROM_NAME` (generic for SaaS apps)
3. `APPBLUEPRINT_Resend__ApiKey` / `APPBLUEPRINT_Resend__FromEmail` / `APPBLUEPRINT_Resend__FromName` (legacy)

## Resend Integration
- Proper configuration following official Resend .NET docs
- `AddHttpClient<ResendClient>()` with `ResendClientOptions`
- `IResend` registered as transient service
- Domain verification required (supports sandbox domain for testing)
