# Changelog

All notable changes to AppBlueprint.Domain will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Domain entities for multi-tenant B2B applications
- User aggregate with strongly-typed UserId
- Team and Organization entities with membership management
- Subscription and billing domain models
- Payment provider integrations (Stripe, Paddle)
- Role-based authorization domain models
- Audit trail and base entity infrastructure
- Domain events infrastructure
- Value objects for email, phone, address
- Strongly-typed IDs using ULID format
- Repository interfaces for domain aggregates
