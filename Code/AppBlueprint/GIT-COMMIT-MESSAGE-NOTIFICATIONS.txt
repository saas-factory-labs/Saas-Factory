## Summary
Implemented a production-ready, multi-channel notification system with Firebase Cloud Messaging (FCM) integration, following Clean Architecture and Domain-Driven Design principles. The system supports real-time in-app notifications via SignalR, push notifications via Firebase, and extensible interfaces for email and SMS.

## Features Added

### Core Notification System
- Multi-channel delivery (InApp, Push, Email, SMS)
- User notification preferences with quiet hours
- Push notification token management (Web, Android, iOS)
- Real-time SignalR notifications with tenant isolation
- Notification read/unread tracking
- Automatic token cleanup on FCM errors

### Domain Layer (Clean Architecture)
- UserNotificationEntity with PrefixedUlid IDs (ntf_)
- NotificationPreferencesEntity with channel toggles
- PushNotificationTokenEntity with device type tracking
- Repository interfaces (INotificationRepository, etc.)
- NotificationType enum (Info, Success, Warning, Error)
- NotificationChannels flags enum for multi-channel selection

### Application Layer
- INotificationService (orchestration with preference checking)
- IInAppNotificationService (SignalR broadcasting)
- IPushNotificationService (FCM token management + sending)
- Request/Response DTOs:
  - SendNotificationRequest (userId, title, message, type, actionUrl, channels)
  - PushNotificationRequest (with Firebase-specific options)
  - RegisterPushTokenRequest (userId, token, platform)

### Infrastructure Layer
- NotificationService: Multi-channel coordinator with quiet hours logic
- InAppNotificationService: SignalR real-time broadcasting
- FirebasePushNotificationService: FCM integration with MulticastMessage API
- NotificationHub: TenantScopedHub<NotificationHub> for multi-tenancy
- Repository implementations with EF Core
- Entity configurations with proper indexes
- Service registration via NotificationServiceRegistration

### UI Components
- NotificationBell.razor: Reusable notification bell component
  - Dropdown with last 10 notifications
  - Unread count badge (99+ for >99 notifications)
  - Real-time SignalR updates
  - Mark as read functionality
  - Relative timestamps (just now, 5m ago, 2h ago)
  - Click-to-navigate for actionable notifications
  - Automatic reconnection handling
- NotificationDemo.razor: Complete demo/test page at /demo/notifications
  - Send test notifications (all channels)
  - Register FCM tokens
  - Real-time notification display
  - Connection status indicator

### Client-Side Firebase Integration
- firebase-messaging-sw.js: Service worker for background push notifications
  - Handles onBackgroundMessage from FCM
  - Shows browser notifications with click handling
  - Service worker lifecycle management
- firebase-messaging-helper.js: JavaScript interop for Firebase SDK
  - Initialize Firebase with config and VAPID key
  - Request notification permissions
  - Get FCM tokens for registration
  - Handle foreground messages

### Database Schema
Created 3 tables via EF Core migration (20260203082406_AddUserNotificationTables):
- user_notifications: Stores in-app notifications with read status
- notification_preferences: Per-user channel settings + quiet hours
- push_notification_tokens: FCM tokens with device type and active status

Indexes added:
- TenantId (multi-tenancy filtering)
- UserId + IsRead composite (unread count queries)
- CreatedAt (chronological ordering)
- Token unique (prevent duplicates)
- UserId + IsActive composite (active token lookups)

## Technical Implementation

### Clean Architecture Compliance
- Domain → Application → Infrastructure → Presentation
- Dependency Inversion: Interfaces in Application, implementations in Infrastructure
- Repository pattern for data access
- Service layer for business logic
- No domain dependencies on infrastructure

### Multi-Tenancy
- All entities implement ITenantScopedEntity
- Automatic tenant filtering in queries
- TenantScopedHub for SignalR tenant isolation
- Compatible with Row-Level Security (RLS)

### Firebase Integration
- FirebaseAdmin 3.0.1 NuGet package (server-side)
- Firebase JS SDK 10.7.1 (client-side)
- Lazy initialization pattern for FirebaseApp
- MulticastMessage API for batch sending (up to 500 tokens)
- WebpushNotification config with VAPID keys
- Error handling with automatic token deactivation

### SignalR Real-Time
- Inherits from TenantScopedHub<NotificationHub>
- User-specific groups (user_{userId})
- Automatic reconnection in NotificationBell component
- Connection state management
- Event-based updates (OnNotificationReceived)

### Security
- HTTPS required for service workers (production)
- JWT-based authentication for SignalR
- Tenant isolation via RLS and query filters
- User authorization (only access own notifications)
- FCM token validation and cleanup
